##  Customer Churn Analysis using Python & SQL Server

##  Project Overview

Customer churn is one of the most critical business challenges for subscription-based companies. Understanding why customers discontinue their subscriptions enables businesses to improve customer retention, reduce revenue loss, and enhance customer satisfaction.

This project presents an end-to-end Customer Churn Analysis solution using **SQL Server** and **Python**. The workflow begins with importing relational data into SQL Server, extracting the data into Python using **PyODBC**, followed by data cleaning, feature engineering, exploratory data analysis (EDA), KPI generation, and business visualization.

The project aims to identify customer churn patterns, analyze customer behavior, measure revenue impact, and generate actionable business recommendations through data-driven insights.

---

#  Business Objectives

The primary objectives of this project are to:

- Measure overall customer churn and retention rates.
- Identify customer segments with the highest churn.
- Analyze churn across subscription plans and geographical locations.
- Evaluate revenue generated by different customer groups.
- Study the relationship between customer support escalations and churn.
- Generate business insights that can improve customer retention strategies.

---

#  Dataset

The project uses three relational datasets imported into SQL Server.

| Table | Description |
|--------|-------------|
| **DB_Customer** | Customer demographic information including customer ID, gender, location, and personal details. |
| **DB_Subscription** | Subscription plans, contract type, monthly charges, cancellation details, churn score, and customer lifetime value. |
| **DB_Support** | Customer complaints, escalation history, CSAT score, and support interactions. |

The tables were joined using **Customer ID** to create a unified dataset for analysis.

# Data Cleaning & Preprocessing

The following preprocessing steps were performed to improve data quality:

- Imported relational data from SQL Server into Python.
- Renamed inconsistent column names.
- Removed duplicate and completely empty records.
- Removed unnecessary columns.
- Converted date columns into datetime format.
- Standardized categorical values.
- Filled missing country values using state-to-country mapping.
- Validated missing values and data types.
- Prepared an analysis-ready dataset.

# Key Insights

- Customers with higher churn scores were significantly more likely to cancel their subscriptions.
- Monthly subscription plans showed higher churn compared to annual plans.
- Customer support escalations were positively associated with customer churn.
- Revenue analysis identified high-value customer segments contributing most to overall business revenue.
- Customer segmentation helped identify high-risk customers for proactive retention campaigns.
- Correlation analysis provided deeper insights into the relationship between customer behavior and churn.

---

# Business Recommendations

Based on the analysis, the following recommendations are proposed:

- Introduce incentives for customers to migrate from monthly to annual subscription plans.
- Proactively engage customers identified as high churn risk.
- Improve customer support processes to reduce escalation rates.
- Monitor churn KPIs through interactive dashboards.
- Focus retention efforts on high-value customer segments.
- Continuously monitor customer feedback and service quality to improve overall customer satisfaction.
